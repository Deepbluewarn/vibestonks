# 마감가 일괄 청산 (시스템 보조금 허용)

금요일 17:00 청산 시 모든 트레이더의 보유 주식은 **그 시점 마감가**로 잔고에 환산된다. 본딩 커브를 따라 순차 청산하지 않고, 모두 동시에 마감가로 평가. 풀이 실제로 보유한 포인트보다 지급액이 많으면 시스템이 차액을 발행하여 보조. 다음 월요일 하드 리셋이 회계상 차이를 모두 0으로 만들어주므로 차액의 누적 문제는 발생하지 않는다.

## Considered Options

- **본딩 커브 순차 청산**: zero-sum 보장, 그러나 청산 **순서**에 따라 손익이 달라져 공정성 시비. 마지막에 청산되는 사람이 항상 손해.
- **마감가 비례 캡 (실제 풀 / 마감가 청구액 만큼만 비례 지급)**: 수학적 보존 완벽, 그러나 "왜 내가 마감가의 93%만 받지?"를 설명해야 함. 토이엔 인지 부담.

## Consequences

- 한 주 동안은 시스템 전체 포인트가 살짝 부풀 수 있음 (양의 합 게임). 이는 의도된 보조금이며 월요일에 사라짐.
- "마지막에 못 팔았다고 손해 본다"는 압박이 없음 → 마감 직전 치킨게임은 약화. 대신 **마감가 자체를 올리는 것**이 모두에게 이득이므로 마감 직전 매도가 감소하는 동기가 됨.
- 안 팔고 가만히 들고 있어도 가격이 오른 만큼 평가받음 → 적극적 매도 인센티브는 "다른 사람이 곧 던질 것 같을 때 먼저 던지기"에 한정.
